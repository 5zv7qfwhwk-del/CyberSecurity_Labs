# Networking Lab Setup (Virtual Environment)

## Overview

This lab documents the setup of my cybersecurity home lab environment,
including virtual machines and initial networking study using Professor
Messer's Network+ content.

The goal is to build a safe environment for practicing network analysis,
security testing, and hands-on labs.

------------------------------------------------------------------------

## Objectives

-   Begin foundational networking study
-   Set up a virtual lab environment
-   Install multiple operating systems for testing and analysis

## Lab Environment Setup

### Virtualization Platform

Installed virtualization software to host multiple operating systems:

-   VirtualBox (Originally my first option but was giving me issues)
-   UTM (I find was a better option for macOS)

Purpose: - Run isolated environments - Simulate real-world network
scenarios - Practice safely without affecting host machine

------------------------------------------------------------------------

### Virtual Machines Installed

#### Kali Linux VM

Kali Linux is a penetration testing and security auditing platform.

Use cases: - Network scanning - Security testing - Packet analysis -
Vulnerability assessment

------------------------------------------------------------------------

#### Windows 10 VM

Windows 10 is used to simulate a target system and analyze logs.

Use cases: - Event log analysis - User activity monitoring - Security
configuration testing

------------------------------------------------------------------------

## Key Takeaways

-   Virtualization allows safe cybersecurity testing environments
-   Multiple operating systems are needed to simulate real-world
    networks
-   Hands-on labs are essential for understanding networking concepts
